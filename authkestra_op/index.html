§Authkestra OP

authkestra-op implements the OpenID Provider (OP) side of OIDC: issuing tokens and running the authorization code grant, as opposed to authkestra-oidc, which consumes an external OP as a relying party.

This crate is intentionally handler-logic-only — it has no dependency on any web framework. authkestra-axum and authkestra-actix (behind an op feature flag) wrap these types into framework-native routes.

§Status

This crate is a skeleton (RFC-003, PR OP.0). No handler logic has landed yet — see docs/rfc-003-oidc-provider.md for the full plan.
